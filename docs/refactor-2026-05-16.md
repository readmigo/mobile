## Conventions established

- Feature modules expose public API via `src/features/<name>/index.ts`; cross-feature consumers always go through that facade
- Same-feature internal references use relative paths, not the `@/features` alias
- Mutation `onError` calls run through `handleApiError(err)`; toast surfaces only when `appError.isUserActionable`
- Auto-save / background mutations do NOT toast — Sentry only
- Mutations don't retry on 5xx (server may have processed); transport failures (NETWORK/TIMEOUT) do retry once
- Business events live in `src/services/analytics.ts` as typed helpers; new event types extend that file rather than calling `trackEvent` directly with magic strings
